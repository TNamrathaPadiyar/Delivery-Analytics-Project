# Delivery Analytics Project

## Overview
This project analyzes delivery performance using shipment-level data to identify delay patterns and operational inefficiencies across couriers, cities, zones, product categories, and time periods.  
The goal is to support data-driven decisions that improve on-time delivery rates and customer satisfaction.

---

## What Was Done
- Loaded and structured raw delivery data into a relational database
- Cleaned data by handling nulls, correcting invalid delays, standardizing courier names, and removing duplicates
- Performed SQL-based analysis to compute average delivery delays across key dimensions
- Visualized trends using Python and Excel dashboards
- Documented insights and recommendations in a detailed analytical report

---

### 5. Dashboard & Reporting
- Built an Excel dashboard summarizing key KPIs
- Created a detailed analytical report explaining trends, business impact, and recommendations

Relevant files:

## Business Value
- Helps identify underperforming couriers and regions
- Enables targeted logistics optimization instead of generic improvements
- Supports proactive planning for seasonal demand surges
- Acts as a decision-support tool for operations and supply chain teams
